    Interface TimeParts

    Represents a human-readable breakdown of a time duration into its fundamental components — days, hours, minutes, and seconds.

    Remarks

    This interface is used by time utility functions such as secondsToParts and partsToSeconds to represent durations in a normalized structure that is easy to read, format, or serialize.

    Each field is an integer number representing the whole count of that unit. None of the fields are fractional, and they are expected to follow these conventional bounds when generated by secondsToParts:

    days    ≥ 0
    hours   ∈ [0, 23]
    minutes ∈ [0, 59]
    seconds ∈ [0, 59]

    However, when constructing manually, these constraints are not enforced by TypeScript — it is up to the user or calling function to maintain validity.

    Example

    // Example 1: Typical decomposition
    const t: TimeParts = { days: 1, hours: 2, minutes: 30, seconds: 45 };

    // Example 2: Used with utility converters
    const total = partsToSeconds(t);   // -> 95445
    const back  = secondsToParts(total); // -> same structure again

    Example

    // Example 3: Display formatting
    const p: TimeParts = { days: 0, hours: 5, minutes: 3, seconds: 9 };
    console.log(`${p.hours}h ${p.minutes}m ${p.seconds}s`);
    // -> "5h 3m 9s"

    See

    • secondsToParts — converts total seconds into this structure.
    • partsToSeconds — converts this structure back into total seconds.
