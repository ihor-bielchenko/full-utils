    splitArrToPortions

    Returns an array of chunks, where each chunk contains at most portionLength elements. The final chunk may contain fewer elements if the input array length is not evenly divisible by portionLength.

    Functions - Conversion

    formatToBool

    Performs loose coercion of different input types into a boolean following predictable rules for numbers, strings, and actual boolean values. Non-recognized values always return false.

    toGB

    Safely normalizes a size value of any supported unit into gigabytes, using binary multiples (1 KB = 1024 bytes, 1 MB = 1024 KB, etc.).

    toGH

    Normalizes a numeric value expressed in various hash rate units (e.g., kH/s, MH/s, TH/s) to gigahashes (GH), using standard decimal scaling (×1000 between adjacent prefixes).

    toH

    Normalizes a numeric value given in kH/s, MH/s, GH/s, TH/s, PH/s, or EH/s to plain hashes per second using decimal scaling (×1000 between adjacent prefixes). Units are case-insensitive and sanitized via normalizeUnit.

    Functions - String

    formatToLowerCase

    Safely transforms an unknown value into a normalized lowercase string. If the input is not a valid non-empty string, the function returns an empty string ("").

    formatToNull

    Safely normalizes optional values by replacing both undefined and blank strings with null, while preserving all other values as-is.
    This helps unify "missing" or "empty" values under a single, database-friendly null representation.

    formatToPhone

    Converts various human-entered phone number formats (with spaces, dashes, parentheses, or local prefixes) into a clean, standardized string beginning with "+" and containing 10–15 digits.

    Returns null if the value is not a valid phone number after normalization.

    formatToTrim

    Safely converts any input into a clean, Unicode-normalized string with all leading/trailing whitespace removed and zero-width characters stripped out.

    Returns an empty string ("") for all non-string inputs.

    formatToUndefined

    Normalizes optional values by replacing both null and blank strings with undefined, while keeping all other values unchanged.
    This is useful when preparing objects for APIs or serialization, where undefined fields are automatically omitted or ignored.

    Functions - Type Guards

    isArr

    A strongly typed wrapper around Array.isArray, with an enhanced TypeScript type guard that asserts the value as a readonly non-empty array (readonly [T, ...T[]]).

    isArrFilled

    A strongly-typed type guard that returns true only if the input value is an array (via isArr) and contains at least one element (length > 0).

    Useful for narrowing unknown data to a readonly non-empty tuple type (readonly [T, ...T[]]), which gives TypeScript safer access guarantees.

    isBool

    A strict type guard that returns true only if value is of type "boolean".

    This helps safely narrow unknown or mixed-type values in TypeScript code, especially when working with dynamic data sources, JSON, or user input.

    isDate

    Returns true if the given value is either:

    • an actual Date instance with a valid timestamp, or
    • a string or number that can be successfully parsed by the JavaScript Date constructor.

    Returns false for invalid dates (Invalid Date), empty strings, NaN, or non-date types.

    isExists

    A minimal and strongly typed utility that acts as a type guard to filter out null and undefined values in TypeScript.
    It is especially useful in array filters and conditional checks where you need to ensure a value is "present".

    isFunc

    A strict type guard that returns true only if value is a callable function.
    It works with regular functions, arrow functions, async functions, class constructors, and built-in functions like setTimeout.

    isNum

    A strict type guard that returns true only if value is of type "number" and is a finite numeric value (not NaN, Infinity, or -Infinity).

    isNumFloat

    Returns true if the value is a finite number and has a fractional part (i.e. not an integer).
    Acts as a strict type guard for float-like numeric values.

    isNumN

    A strict type guard that returns true only if the input is a finite number and its value is less than 0.
    Useful for safely detecting negative numeric values without coercion.

    isNumNZ

    A strict type guard that returns true if the input is a finite number and its value is negative or zero.
    Commonly used to identify non-positive numeric values.

    isNumP

    A strict type guard that returns true only if the input is a finite number greater than zero (> 0).
    Useful for safely validating numeric values that must be positive — such as counters, IDs, amounts, or dimensions.

    isNumPZ

    A strict type guard that returns true if the input is a finite number and its value is non-negative (≥ 0).
    Useful for safely validating values such as counters, sizes, durations, and other quantities that cannot be negative.

    isObj

    A strict type guard that returns true only if the input is a plain object created using an object literal ({}) or Object.create(null).
    Excludes arrays, functions, null, and instances of custom classes.

    isObjFilled

    A strict type guard that returns true only if the input is a plain object (validated by isObj) and has at least one own enumerable key.
    In other words, it must be an object literal like { a: 1 }, not {}.

    isStr

    A strict type guard that returns true only if the provided value has the JavaScript type "string".
    Rejects all non-string types, including String objects created via new String().

    isStrFilled

    A strict type guard that returns true only if the value is a string containing at least one non-whitespace character.
    This is an extended form of isStr that excludes empty ("") and whitespace-only strings (" ").

    Functions - Validation

    isEmail

    Validates that the input is a non-empty string and matches a simplified but practical email pattern according to RFC 5322-compatible syntax rules.

    isIpAddr

    Validates that the input is a string containing four octets (e.g. "192.168.0.1"), where each octet is a number between 0 and 255, separated by dots.

    isMacAddr

    Validates that the input is a string formatted as a standard 6-octet MAC (Media Access Control) address, consisting of 12 hexadecimal digits separated by either colons (:) or hyphens (-).

    isPassword

    A flexible password validation helper that checks for length, uppercase and lowercase letters, digits, and special characters.
    All rules are configurable through the optional PasswordOptions parameter.

    isPhone

    A flexible, language-neutral phone number validator that accepts both string and numeric inputs.
    It supports optional leading "+", allows dashes ("-") as separators, and ensures that the number structure is syntactically valid — not necessarily region-specific.

    isStrAscDesc

    A strict type guard that validates sorting direction strings —
    "asc" (ascending) or "desc" (descending).
    Useful for validating API parameters, SQL sort directives, or UI sort controls.

    isStrBool

    A strict type guard that returns true if the input is a non-empty string equal to "true" or "false" (case-insensitive, with whitespace ignored).
    Commonly used for validating query parameters, environment variables, or serialized booleans.

    isVar

    A strict type guard that returns true only if the value is a string matching the pattern of a valid programming variable name:
    must start with a letter (A–Z, a–z) or underscore (_), and may contain only letters, digits, or underscores afterward.
