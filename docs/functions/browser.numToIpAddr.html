    Function numToIpAddr

    • numToIpAddr(num: number): string

      Converts a 32-bit unsigned integer (numeric IPv4 representation) back into its dotted-decimal string form (e.g. "192.168.0.1").

      Parameters

      • num: number

        The 32-bit unsigned integer representing an IPv4 address.

      Returns string

      A dotted-decimal IPv4 string (e.g. "10.0.0.1"),
      or an empty string if the input is invalid.

      Remarks

      This is the inverse of ipAddrToNum.
      It interprets the input number as a big-endian (network-byte-order) IPv4 value, extracting each of the four octets and joining them into the standard dotted-quad notation.

      If the input is not a valid finite number (checked via isNumP), an empty string "" is returned instead of throwing an exception.

      The resulting string always consists of exactly four decimal octets separated by dots, with each octet in the range 0–255.

      • Internally uses DataView to ensure consistent big-endian behavior across all platforms.
      • The output format is always normalized (no leading zeros, no spaces).
      • For the forward direction (string → number), see ipAddrToNum.

      Example

      // Example 1: Basic conversion
      numToIpAddr(3232235521);
      // -> "192.168.0.1"

      // Example 2: Edge values
      numToIpAddr(0);          // -> "0.0.0.0"
      numToIpAddr(4294967295); // -> "255.255.255.255"

      Example

      // Example 3: Invalid inputs
      numToIpAddr(NaN);         // -> ""
      numToIpAddr(Infinity);    // -> ""
      numToIpAddr(-5);          // -> ""

      Throws

      Never throws; invalid inputs simply return an empty string.

      See

      • ipAddrToNum — converts dotted IPv4 strings to numeric form.
      • parseIPv4 — alternative parser using bitwise arithmetic.
