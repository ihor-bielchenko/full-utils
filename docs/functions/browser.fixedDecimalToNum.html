    Function fixedDecimalToNum

    • fixedDecimalToNum(value: FixedDecimal): number

      Converts a FixedDecimal — an exact decimal representation — into a native JavaScript number (IEEE 754 double precision).

      Parameters

      • value: FixedDecimal

        The FixedDecimal instance to convert. Must contain:

        • sign: either 1 or -1;
        • digitsInteger: a bigint representing all digits without any decimal point;
        • scale: how many digits belong after the decimal point.

      Returns number

      The approximate numeric value as a JavaScript number. If the string form is too large or too precise, the result may be rounded or become Infinity.

      Remarks

      This function is a lossy conversion when value contains more precision than JavaScript’s floating-point format can represent. It internally calls fixedDecimalToStr to produce a normalized string such as "-123.4567" and then passes it to the built-in Number() constructor.

      O(n) relative to the number of digits in digitsInteger (due to string creation in fixedDecimalToStr).

      Example

      const fd: FixedDecimal = { sign: 1, digitsInteger: 12345n, scale: 2 };
      // Represents exactly 123.45
      const num = fixedDecimalToNum(fd); // 123.45 (Number)

      Example

      // Handles negative numbers as well
      const neg: FixedDecimal = { sign: -1, digitsInteger: 987n, scale: 3 };
      const n = fixedDecimalToNum(neg); // -0.987

      Example

      // Extreme precision may lose digits beyond ~15–17 significant figures
      const big: FixedDecimal = { sign: 1, digitsInteger: 12345678901234567890n, scale: 10 };
      const approx = fixedDecimalToNum(big);
      console.log(approx); // ~1234567890.1234567  (rounded)
