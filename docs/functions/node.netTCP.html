    Function netTCP

    • netTCP(message: string, host: string, options?: NetTCPOptions): Promise<string>

      Open a TCP connection to a host, send a text payload, and return the server’s response as a string.

      Parameters

      • message: string

        The request body to send. If lineTerminator is set, it is appended automatically.

      • host: string

        Hostname or IP address to connect to.

      • options: NetTCPOptions = {}

        Behavior and safety knobs for the connection, timeouts, encoding, and buffering.

        Options controlling TCP connection behavior, request formatting, and response handling.

        • OptionalconnectTimeoutMs?: number

          Separate timeout for the connect phase (DNS+TCP handshake). If the connection is not established within this period, the operation fails.

          Default Value

          0 (no connect timeout)

          Remarks

          On expiry, rejects with ETIMEDOUT ("Connect timed out").

        • Optionalencoding?: "utf8" | "utf-8"

          Text encoding used to encode the outgoing message and decode the response.

          Default Value

          'utf8'

          Remarks

          The accepted values are 'utf8' or 'utf-8' (normalized to 'utf8').

        • OptionalhalfCloseAfterWrite?: boolean

          When true, call socket.end() immediately after the request payload is fully written. This is useful for protocols where the client signals “no more data” by half-closing.

          Default Value

          false
        • OptionallineTerminator?: "" | "\n" | "\r\n"

          Optional line terminator automatically appended to the outgoing message.

          Default Value

          '' (no terminator)

          Remarks

          Common values are '\n' or '\r\n' for CRLF-terminated protocols.

        • OptionalmaxBytes?: number

          Hard cap on response size in bytes. If the cumulative number of bytes read exceeds this limit, the read is aborted.

          Default Value

          0 (no limit)

          Remarks

          On overflow, rejects with ERESPONSE_TOO_LARGE.

        • Optionalport?: number

          Remote TCP port to connect to.

          Default Value

          4028

          Remarks

          Must be an integer in the range 1–65535, otherwise EBADPORT is thrown.

        • Optionalsignal?: AbortSignal

          Cooperative cancellation token. If the signal is already aborted (or aborts later), the operation is terminated early.

          Remarks

          Rejects with EABORT ("Operation cancelled").

        • OptionaltimeoutMs?: number

          Idle timeout in milliseconds applied to the underlying socket. If no data is received within this period after connection, the operation fails.

          Default Value

          0 (no idle timeout)

          Remarks

          On expiry, the promise rejects with ETIMEDOUT ("Idle timeout").

      Returns Promise<string>

      A promise that resolves to the full response string (decoded using encoding).

      Throws

      ENODEONLY Thrown when executed in a non-Node.js environment (e.g., a browser).

      Throws

      EBADHOST Thrown when host is not a non-empty string.

      Throws

      EBADMSG Thrown when message is not a non-empty string.

      Throws

      EBADPORT Thrown when port is outside the range 1–65535 or not an integer.

      Throws

      ENETIMPORT Thrown if dynamic import of the node:net module fails.

      Throws

      EABORT Thrown if the provided AbortSignal is already aborted or aborts during the operation.

      Throws

      ETIMEDOUT Thrown in these cases:

      • "Connect timed out": connect phase exceeded connectTimeoutMs.
      • "Idle timeout": no data within timeoutMs after connection.

      Throws

      ERESPONSE_TOO_LARGE Thrown when the accumulated response exceeds maxBytes.

      Throws

      Any socket 'error' event is forwarded (e.g., ECONNREFUSED, ENOTFOUND, EPIPE).

      Remarks

      • Lifecycle: The socket’s listeners are cleaned up and the socket destroyed on settle.
      • Close semantics: If the socket emits 'close' with hadError=true, the promise rejects with ECLOSE.
      • Performance: If you expect large responses, set a realistic maxBytes to avoid excessive memory usage.
      • Protocols: For request/response style text protocols (e.g., simple line-based services), set lineTerminator (e.g., '\n') and optionally halfCloseAfterWrite=true if the server expects EOF to start responding.

      Example

      // CRLF-terminated request with half-close to signal EOF
      const resp = await netTCP('STATUS', '192.0.2.10', {
        port: 9000,
        lineTerminator: '\r\n',
        halfCloseAfterWrite: true,
        timeoutMs: 3000,
        maxBytes: 256 * 1024, // 256 KiB cap
      });
      console.log(resp);
