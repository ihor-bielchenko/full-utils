    Function rangeIPv4ToArr

    • rangeIPv4ToArr(from: string, to?: string, opts?: RangeIPv4Options): string[]

      Materializes an IPv4 range into an array of dotted-quad strings.

      Parameters

      • from: string

        See rangeIPv4 for accepted forms (CIDR or IPv4).

      • Optionalto: string

        Optional end address (non-CIDR mode). Ignored for CIDR input.

      • opts: RangeIPv4Options = {}

        Options including the array size limit. See RangeIPv4Options.

      Returns string[]

      A new array with all IPv4 addresses in the range (inclusive).

      Remarks

      This is a convenience wrapper around the lazy rangeIPv4 generator that collects the yielded addresses into a new array. To protect against excessive memory usage, a hard limit is enforced (default 1_000_000). If the range exceeds the limit, an error is thrown and no partial array is returned.

      Prefer using the generator for very large ranges, streaming the results into your processing pipeline (e.g. writing to a file or socket).

      Example

      // Small range OK
      rangeIPv4ToArray('192.168.1.10', '192.168.1.13');
      // -> ['192.168.1.10','192.168.1.11','192.168.1.12','192.168.1.13']

      // Will throw if exceeds the limit
      rangeIPv4ToArray('10.0.0.0/16', undefined, { limit: 10_000 }); // Error

      Throws

      If the realized array would exceed opts.limit.

      See

      rangeIPv4 for lazy iteration without materialization.
